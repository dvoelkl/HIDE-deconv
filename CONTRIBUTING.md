# Contributing to HIDE-deconv

We welcome contributions to this project.

## Questions
If you have questions about the project, the workflow or possible contributions, please open an [GitHub Issue](https://github.com/dvoelkl/HIDE-deconv/issues) and use a clear and descriptive title.

Please include:
- a short description of your question
- the relevant context or workflow
- any files, command or error messages that may help us answer it

## Issues
If you find a bug or unexpected behavior, please open a [GitHub Issue](https://github.com/dvoelkl/HIDE-deconv/issues) and use the bug label.

Please include:
- a clear description of the problem
- the exact workflow that triggered it
- the expected and the actual behavior
- relevant input data, if needed to reproduce the issue

We try to resolve issues as soon as possible, usually within 5 working days.

## Feature requests
If you have an idea for an additional feature, please open a [GitHub Issue](https://github.com/dvoelkl/HIDE-deconv/issues) and use the enhancement label.

Please include:
- a short description of the proposed feature
- a drafted workflow
- a short explanation of why the feature would be useful

We will review the request and discuss it within the team.

## Pull requests
If you want to contribute code directly, please open a pull request.

Before submitting a pull request, please make sure that:
- the Code of Conduct is respected
- your changes are focused and consistent with the project style
- tests are included for new behavior or bug fixes
- existing tests pass locally
- the change is described clearly in the pull request

If your change affects user-facing behavior, please add a short explanation of the workflow or output that changed.

## General notes
- Please keep changes small and easy to review.
- If you use GenAI for coding, please ensure that the code remains human-understandable and not unnecessarily complex. Please also ensure that no copyright is violated.
- If possible, include reproducible examples for bugs.
- If your contribution depends on larger changes, open an issue first so we can discuss the approach.
